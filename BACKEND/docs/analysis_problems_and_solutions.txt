1) CRITICAL — `nexura/history_db.py` get_session indent xatosi
   - Muammo: `get_session` funksiyasida `if not row:` dan keyin `return None` noto'g'ri joylashgan va blok indentatsiyasi buzilgan. Bu modul import vaqtida SyntaxError yoki logic xatosi keltirib chiqarishi mumkin.
   - Risk: serverni ishga tushirishni bloklashi, tarix DB operatsiyalar ishlamasligi.
   - YeChIM: `get_session` ichidagi indent va bloklarni to'g'irlash. Quyidagi qatorlar tuzatilsin (soddalashtirilgan):
       if not row:
           return None
       session = dict(row)
       ... (keylarni JSON pars qilib qaytarish)
   - Tavsiya: unit test yozish — `HistoryDB.get_session` uchun.

2) HIGH — Pydantic modellarda mutable default qiymatlar
   - Muammo: `nexura/models/schemas.py` da list default qiymatlar bevosita `[]` sifatida berilgan (e.g., `ports: list[PortInfo] = []`). Bu model instansiyalari orasida qiymatlar aralashishiga olib keladi.
   - Risk: ma'lumotlar kross-contamination, noaniq xatoliklar runtime’da.
   - YeChIM: pydantic v2 uchun `Field(default_factory=list)` yoki `list` tipida aniq default_factory ishlatish. Misol:
       ports: list[PortInfo] = Field(default_factory=list)
   - Tavsiya: hamma BaseModel-lardagi mutable defaultlarni tekshiring (ports, vulnerabilities, results, templates). Unit test.

3) HIGH — Noto'g'ri maydon nomi `cvss_score` vs `cvss`

   - Risk: TypeError yoki maydon qabul qilinmasligi (missing field yoki noto'g'ri serializatsiya).
   - YeChIM: Bir xil nomga keltirish — tavsiya: `cvss` saqlang (modelsga moslash). `web/app.py` va `history_db.py` dan `cvss_score` o‘rniga `cvss` ishlatilsin.
